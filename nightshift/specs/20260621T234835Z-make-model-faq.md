# Spec — Per-model FAQ blocks + FAQPage JSON-LD on make+model pages

**Slug:** make-model-faq
**Lane:** [goal] — `[P2][goal] Per-model FAQ blocks + FAQPage JSON-LD` (Brainstorm
2026-06-20, section A. Page quality / crawl-efficiency). Last non-bug cycle
(`aircraft-filter-ranges`) pulled `[want]`, so `[goal]` is owed. STAGE=INDEXING →
unique content depth + structured data make the priority for-sale pages genuinely
index-worthy instead of templated boilerplate Google skips.

## Goal
Add 3 genuine, model-specific Q&As + valid `FAQPage` JSON-LD to each of the 20
curated `/aircraft/[make]/[model]` pages, so each page carries unique evergreen
prose (less templated) and is structured-data-eligible.

## Scope (small, additive)
- `src/lib/seo.ts` — add optional `faqs?: { q: string; a: string }[]` to the
  `SeoMakeModel` type; author 3 real Q&As for each of the 20 curated entries
  (drawn from the existing `specs`/`costToOwn` prose + general GA knowledge — NO
  fabricated stats, NO live counts in the text so answers stay evergreen).
- `src/lib/aircraftJsonLd.ts` — add `buildFaqPageJsonLd(faqs, { url, name })`
  returning a `FAQPage` schema object (or `null` when no faqs).
- `src/components/ModelFaq.tsx` — new presentational component rendering the FAQ
  visually (native `<details>`/`<summary>` accordion, card style matching
  `ForSaleGuideLinks`). Visible text == JSON-LD text (Google parity requirement).
- `src/app/aircraft/[make]/[model]/page.tsx` — render `<ModelFaq>` + inject the
  FAQPage JSON-LD, only when the resolved entry has `faqs` (curated combos only;
  dynamically-discovered combos render nothing — graceful).

## Acceptance criteria
- [ ] `npx next build` + typecheck pass.
- [ ] A curated make+model page (e.g. `/aircraft/cessna/172`) renders a visible
      "Frequently asked questions" section with 3 model-specific Q&As.
- [ ] Served HTML for that page contains one parseable `application/ld+json`
      FAQPage block whose questions/answers match the visible text 1:1.
- [ ] FAQ answers contain no live counts and no fabricated numeric stats (evergreen).
- [ ] QA smoke passes desktop 1280 + mobile 375 (HTTP 200, no app console errors,
      no horizontal overflow); screenshots look right (FAQ readable, no overlap).
- [ ] No visual/layout regression to the rest of the page; existing JSON-LD blocks
      (ItemList, AggregateOffer) still parse.

## Out of scope
- Partnership / airport / state pages (make+model only this cycle).
- FAQs for dynamically-discovered (non-curated) combos.
- Any schema/DB/SQL change; any new color/palette; nav/IA changes.
