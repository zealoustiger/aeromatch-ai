# Internal linking graph — SLICE 2: family→family rails

**Lane:** [goal] · **Scoreboard at orient:** 54

## Goal
Add crawlable `<Link>` "related" rails (real anchors, server-rendered, not JS) on the two new
aircraft SEO page families so crawl equity + link discovery flow laterally across the whole
programmatic surface (make+model ↔ geo state), and users can navigate sideways.

## Scope (small, additive)
- `src/components/AircraftSaleList.tsx` — add two cheap server-side aggregation helpers:
  - `topStatesForMakeModel(make, modelPattern, notModelPattern, limit)` → `{ code, n }[]` for the
    states (USPS code, non-null) with the most ACTIVE listings of that family. Each returned state
    has ≥1 listing ⇒ its `/aircraft/for-sale/[state]` page resolves 200 (no 404).
  - `topMakeModelsForState(code, limit)` → resolves each active listing's raw make/model in that
    state through `resolveMakeModelFamily` (the existing single source of truth), aggregates by the
    resolved combo, and returns the top families that have a REAL page. Skips anything that doesn't
    resolve to an existing combo (null models, non-manufacturer makes), so every link 200s.
- `src/app/aircraft/[make]/[model]/page.tsx` — add a "More {Make} models for sale" rail (same-make
  inventory-backed combos from `getInventoryMakeModels()`, excluding current) + a "Browse {Model}
  by state" rail (from `topStatesForMakeModel`). Keep/extend the existing related-combos rail.
- `src/app/aircraft/for-sale/[state]/page.tsx` — add a "Popular aircraft for sale in {State}" rail
  (from `topMakeModelsForState`, linking to make+model pages). Keep the existing other-states rail.

## Acceptance criteria
1. `/aircraft/[make]/[model]` (e.g. cessna/172) renders: (a) a same-make "More Cessna models for
   sale" rail of real `<Link>`s to other inventory-backed combos of that make, and (b) a "Browse
   Cessna 172 by state" rail of real `<Link>`s to `/aircraft/for-sale/[state]` pages.
2. `/aircraft/for-sale/[state]` (e.g. california) renders a "Popular aircraft for sale in
   California" rail of real `<Link>`s to make+model pages that EXIST.
3. EVERY rail link is a server-rendered `<a href>` (crawlable), never points to the current page,
   and resolves HTTP 200 (spot-check 3–4 across both pages — none 404).
4. `npx next build` green + `npx tsc --noEmit` (no NEW errors beyond the pre-existing 3 `.test.ts`
   import-extension ones).
5. No console/hydration errors and zero horizontal overflow at 375px on both pages.
6. Sky-blue accent only (no emerald/new palette). Compact rails, no thin/doorway text.

## Out of scope
- No schema change (read-only over existing `aircraft_for_sale`). No migration.
- No change to titles/meta/canonical/JSON-LD/breadcrumbs/AlertSignup.
- No new page families; no sitemap change. No filter UI work.
- No links to combos/states without a live page (guardrail: link only to pages that exist).
