# Spec — Signed-in indicator + profile menu

**Slug:** profile-menu
**Lane:** [want] — `[P2][want] Signed-in indicator + profile menu` (Identity & account, BACKLOG)
**Date:** 2026-06-22

## Goal
Give signed-in users a clear identity indicator (avatar + email) in the top-right and a
single profile dropdown that consolidates the account-related links — decluttering the
crowded desktop nav, where today the signed-in state is only implied by a spray of extra
links and a bare "Sign out" button with no sense of *who* is signed in.

## Scope (small)
- New `src/components/ProfileMenu.tsx` — a client dropdown: an avatar button (deterministic
  colored circle with the user's initial, since users have no photo yet) that opens a menu
  with the signed-in email as a header + links to Messages, Saved, My Searches, Admin (admins
  only), and a Sign out action. Click-outside + Escape close it.
- `src/components/Nav.tsx` — on **desktop**, when signed in, move Messages / My Searches /
  Saved / Admin out of the center nav bar and into the new ProfileMenu (rendered in the
  right-actions cluster, replacing the inline "Sign out" button). Keep the "Post a Listing"
  CTA. Logged-out desktop unchanged (still shows "Sign in"). On **mobile**, add a small
  user-identity header (avatar + email) at the top of the existing mobile menu panel when
  signed in; the existing mobile links stay as-is.

## Acceptance criteria
- Signed in (desktop): top-right shows an avatar (initial in a colored circle); clicking it
  opens a dropdown showing the user's email + Messages, Saved, My Searches, Sign out (and
  Admin only for admin emails). Clicking outside or pressing Escape closes it.
- The center desktop nav no longer carries Messages / My Searches / Saved / Admin when signed
  in — those now live in the profile dropdown (less clutter). Base links (Partnerships, Planes
  for Sale, Tools, Guides, About) unchanged.
- Logged-out desktop nav is unchanged (Sign in + Post a Listing).
- Mobile: signed-in shows an avatar + email header atop the menu; all existing mobile links
  still work. Logged-out mobile unchanged.
- Sign out still works (signs out, routes home, refreshes) from both the dropdown and mobile.
- `next build` + typecheck pass; QA smoke (HTTP 200, no app console errors, no horizontal
  overflow) passes at 1280 + 375 on `/` and `/aircraft`; screenshots look right.

## Out of scope
- Real cartoon/pilot avatar generation (this slice uses an initial-based avatar; cartoon
  avatars are a follow-up slice).
- An `/account` settings page (separate backlog item; not built here — dropdown links only to
  existing routes).
- Any change to auth flows, `src/app/auth/**`, or session handling (read-only auth state, as today).
