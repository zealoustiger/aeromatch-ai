# Spec — /guides hub/index page

## Goal
Stand up a real `/guides` hub page that lists the three existing content guides with title + description + crawlable links, so the three pillars cross-promote and the guides' "Guides" breadcrumb becomes a real link.

## Scope (files expected to touch)
- NEW `src/app/guides/page.tsx` — the hub: H1, intro, 3 guide cards (Link each), unique Metadata (title/description/canonical mirroring the guide pages), CollectionPage+ItemList JSON-LD + BreadcrumbList (Home › Guides via existing `Breadcrumbs`).
- EDIT `src/app/guides/aircraft-co-ownership/page.tsx` — Breadcrumbs "Guides" crumb gets `href: '/guides'`.
- EDIT `src/app/guides/cost-of-aircraft-co-ownership/page.tsx` — same.
- EDIT `src/app/guides/aircraft-partnership-agreement/page.tsx` — same.
- EDIT `src/app/sitemap.ts` — add `/guides` to the Guides static block.
- EDIT `src/components/Footer.tsx` — add an "All guides" hub link at the top of `guideLinks`.

## Acceptance criteria
- `/guides` renders: H1 ("Aircraft Co-Ownership Guides"), an intro paragraph, and 3 cards each with the guide title + one-line description, each a real `<Link>` to the guide route (resolves 200).
- Unique `<title>`, meta description, and `alternates.canonical` (mirrors the guide pages' Metadata shape).
- CollectionPage + ItemList JSON-LD listing the 3 guides (3 itemListElement) AND BreadcrumbList JSON-LD (Home › Guides) — all parse as valid JSON.
- All three guide pages' "Guides" breadcrumb is now a crawlable `<Link href="/guides">` (rendered as `<a>`).
- `/guides` is in `sitemap.xml`; Footer Guides group has an "All guides" link at the top.
- Build green + tsc clean (only the 3 known baseline `.test.ts` errors). Zero horizontal overflow at 375px; no console errors; sky-blue accent only.

## Out of scope
- No schema/DB change. No new guide content (guide 4). No changes to the guide bodies beyond the breadcrumb href. No nav-bar change.
