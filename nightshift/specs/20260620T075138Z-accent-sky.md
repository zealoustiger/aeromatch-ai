# Spec — Brand-accent consistency sweep: emerald → sky-blue (user-facing)

## Goal
Standardize stray `emerald` accent/CTA/branding colors on user-facing partnership +
auth pages/components to the app's canonical sky-blue accent, enforcing the taste
rule of ONE accent color (no new palettes).

## Why (lane / metric)
Lane **[want]** (taste/brand consistency). Scoreboard at orient = **54** (scoreboard
prints 60 pageviews/7d). Prior non-bug cycle pulled `[goal]` (family-rails), so the
1:1 alternation correctly pulls `[want]` this cycle.

## Canonical sky accent (mirror these — do NOT invent shades)
Source of truth, already used app-wide (e.g. `/partnerships/[id]` contact card):
- Primary button: `bg-sky-600 hover:bg-sky-700` (+ white text)
- Secondary/outline button: `border-sky-300 ... text-sky-700 hover:bg-sky-50`
- Accent icon: `text-sky-500` / `text-sky-600`
- CTA/info card: `border-sky-200 bg-sky-50`, heading `text-sky-800`, body `text-sky-700`
- Data badge: `bg-sky-50 text-sky-700 ring-sky-200` (and `bg-sky-100` variant)
- Form focus ring: `focus:ring-sky-100` (canonical app pattern; replaces `focus:ring-emerald-200`)
- Checkbox accent: `text-sky-600`

## Scope (files to touch — presentational only)
- src/app/auth/page.tsx (success check icon)
- src/app/partnerships/page.tsx (header icon)
- src/app/partnerships/seeking/page.tsx (header icon + "Post a listing" CTA)
- src/app/partnerships/seeking/[id]/page.tsx (hours badge, "Have a plane that fits?" card + 2 buttons)
- src/components/SeekerCard.tsx (hours badge)
- src/components/CaptureClient.tsx (success check icon)
- src/components/PostSeekerListingForm.tsx (submit button)

## Acceptance criteria
1. `grep -rn emerald` over the 7 scoped files returns nothing.
2. `npx next build` green + typecheck clean (no NEW errors vs baseline).
3. On `/partnerships`, `/partnerships/seeking`, `/partnerships/seeking/[id]`, `/auth`
   (PRODUCTION build via `npm run start`) every former-emerald accent now renders
   sky-blue; no stray emerald button/link/icon/badge remains on those pages.
4. Buttons remain legible, hover states work, contrast OK; no layout/copy change.
5. No horizontal overflow at 375px; no console/hydration errors.

## Out of scope
- `src/app/admin/**` (internal — explicitly excluded).
- `globals.css` emerald palette token DEFINITIONS — leave them (harmless unused
  tokens; removing risks breaking any non-scoped emerald still present elsewhere).
- Other components still using emerald that are NOT on the four QA routes
  (PartnershipCard share-fraction badge, AircraftSaleCard, AircraftSaleFilters,
  SignUpGate, FeedbackWidget, DraftCard, SaveSearchButton) — out of this cycle's
  reviewable scope; can be a follow-up cycle.
- No layout/logic/copy changes. NO schema change. NO new palette colors.

## Judgment note
None of the 7 scoped emeralds carry a DISTINCT semantic green meaning
(no "verified/available/success-vs-error" contrast where green is the signal).
The check-circle icons are confirmation accents and the hours pills are decorative
data badges sitting beside neutral slate badges — all are brand accent, so all
convert to sky. Nothing left green in scope.
