# Trust ranking — SLICE 2 of the [P1][want] Listing trust layer

## Goal
Make the same trust/completeness signals that slice 1 made *visible* on `/partnerships`
also AFFECT ORDER: float more-complete / on-platform / real-photo / member-posted listings
above thin / off-platform ones in the default `/partnerships` view — without regressing
filters, search relevance, or any explicit sort.

## Scope (small, additive, reversible — NO schema change)
- `src/components/PartnershipList.tsx` — after fetching + filtering, apply a stable
  secondary sort: trust score (from slice 1's `evaluateTrust`) DESC, with the existing
  recency order (`created_at` DESC, already applied by the DB query / mock array order)
  preserved as the tie-breaker via stable `Array.prototype.sort`. Applied to BOTH the
  Supabase path and the mock-data path so behavior matches.
- `src/lib/partnershipTrust.test.ts` — add a small unit test asserting that a sort by
  trust score reorders a thin listing below a complete one (and ties keep input order).

## Reuse (do NOT duplicate signals)
- Use `evaluateTrust(p).score` from `src/lib/partnershipTrust.ts` as the sort key — the
  EXACT helper the badge uses, so the badge count and the ranking can never drift.

## Acceptance criteria (QA grades against these)
1. On `/partnerships` default view, listings with higher trust-badge counts appear before
   lower ones — badge counts trend high→low top-to-bottom.
2. Filters/search still work (airport, state, make, share_type, max_monthly, max_buyin) and
   the result COUNT is unchanged vs. before the sort (no items dropped/added).
3. Among listings with equal trust score, the prior recency ordering is preserved (stable).
4. `npx next build` + typecheck pass; the partnershipTrust unit tests pass (incl. the new one).
5. Mobile-first 375px: no layout change, no overflow, sky-blue accent unchanged; no console
   or hydration errors at desktop + 375px.

## Out of scope (DEFER)
- Slice 3: poster completion nudges.
- Slice 4: reducing off-platform redirects.
- Any schema/migration change. Any change to the trust signal definitions or the badge UI.
- Changing the primary intent of explicit filters/search relevance.
