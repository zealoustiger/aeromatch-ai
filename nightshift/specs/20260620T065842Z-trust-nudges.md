# Trust nudges — slice 3 of the Listing trust layer ([P1][want])

## Goal
On their OWN partnership listing detail page, nudge a signed-in owner to complete
the listing by naming exactly which trust signals are missing (real photo /
complete specs / on-platform contact) with a link to the listing-management
surface — reusing the SAME `evaluateTrust` helper from slice 1 as the single
source of truth for what's missing.

## Scope (small, additive)
- NEW `src/components/ListingOwnerNudge.tsx` — presentational client component.
  Given the partnership, computes the unmet signals via `evaluateTrust(p)` and
  renders a tasteful sky-blue inline card naming each missing item + a link to
  edit/manage. Renders nothing when nothing is missing.
- EDIT `src/app/partnerships/[id]/page.tsx` — server-side owner check (compare
  `auth.getUser().id` to `p.poster_id` using the EXISTING `createServerSupabaseClient`),
  render `<ListingOwnerNudge>` ONLY when the viewer is the owner. (READ-only use of
  the frozen supabase-server client — never edited.)

## Acceptance criteria (QA grades against these)
1. Build green: `npx next build` + `npx tsc --noEmit` (touched files) pass.
2. The nudge NEVER renders for a logged-out visitor on a listing detail (no leak)
   — verified live in the browser.
3. The nudge NEVER renders for a signed-in non-owner (owner-gate by `poster_id`).
4. When it renders (owner, missing signals), it names EXACTLY the unmet signals
   from `evaluateTrust` — no redefinition of the signals; member-posted is omitted
   from the actionable list (it's intrinsic to being the poster, not an action).
5. Sky-blue, mobile-first 375px, no horizontal overflow, no dark patterns / fake
   urgency / modal; an inline card on the owner's own detail page.
6. No new console errors or hydration warnings at desktop + 375px.

## Out of scope / deferred
- No schema change (compute from existing fields only).
- No new edit route/flow this cycle. There is NO existing per-listing edit route
  (only `/partnerships/new`), so the nudge links to the most sensible existing
  owner surface and the edit-route gap is noted in CHANGELOG `Next:`.
- Slice 4 (reduce off-platform redirects) — deferred, noted in `Next:`.
- Do NOT touch auth/login, admin gating, or the listing CREATE pipeline logic.
