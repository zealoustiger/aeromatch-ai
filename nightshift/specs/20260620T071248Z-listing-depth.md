## Goal
Give the partnership detail page (`/partnerships/[id]`) real listing depth: a multi-photo
gallery (with graceful single-photo / placeholder fallback) and a crawlable "Similar
partnerships" rail of real other listings — adapting the existing components from
`feat/listing-depth` (PR #18), not rebuilding.

## Acceptance criteria (what QA grades against)
1. `npx next build` compiles + typechecks; `npx tsc --noEmit` clean for touched files (pre-existing `.test.ts` import-extension errors excepted).
2. **Multi-photo listing** (`3f556db0…` Mooney, 3 images) renders a gallery: main image + a thumbnail strip; the `N / total` counter shows; thumbnails switch the main image. No console/hydration errors. Lightbox opens on click and closes on Escape/click-out.
3. **Single-photo listing** (`8b769b19…` Cessna 182, 1 image) degrades gracefully: one image, **no** thumbnail strip, **no** broken carousel/counter.
4. **Zero-photo listing** (`b6296989…`) shows the make-based placeholder image with the existing **"Not actual plane photo"** badge — matching the rest of the app's placeholder behavior. No broken/empty image.
5. **Similar rail** shows real OTHER partnership cards (excludes the current listing), each a crawlable `<Link>` to `/partnerships/<other-id>` that resolves **HTTP 200** (not the current id, not 404). Rail renders nothing when there are no sensible matches.
6. **No 375px horizontal overflow** (`scrollWidth === clientWidth === 375`) on the detail page with the gallery + rail present; sky-blue accent; no autoplay/dark patterns. Desktop + 375px screenshots captured.
