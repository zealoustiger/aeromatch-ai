# Make-level FAQ blocks + FAQPage JSON-LD on `/aircraft/[make]`

**UTC:** 2026-06-22T00:08Z · **slug:** `make-faq` · **lane:** `[goal]`

## Goal
Give each curated make hub page (`/aircraft/[make]` — Cessna, Piper, Cirrus,
Beechcraft, Mooney, Diamond, Van's, Grumman) a real "Frequently asked questions"
section — 3 genuine, evergreen, make-level Q&As rendered as a no-JS accordion AND
emitted as `FAQPage` structured data (visible text matching the JSON-LD 1:1) — to add
unique content depth + rich-result eligibility on these priority INDEXING-stage pages.
Directly extends the 2026-06-21 `make-model-faq` cycle's "Next" to the make level,
reusing the exact same component + helper.

## Scope (small, additive)
- `src/lib/seo.ts` — add `faqs?` to the `SeoMake` type; add a `MAKE_FAQS` map keyed by
  `makeSlug` (8 curated makes only); attach it in `resolveMake` (mirrors how
  `getMakeModel` attaches `MODEL_FAQS`) without mutating the source.
- `src/app/aircraft/[make]/page.tsx` — import `buildFaqPageJsonLd` + `ModelFaq`; emit
  the `FAQPage` JSON-LD script and render `<ModelFaq label={entry.make} faqs={entry.faqs} />`
  near the bottom (only when `entry.faqs` is present).

No new component, no new color, no new dependency, NO schema/DB/SQL.

## Acceptance criteria
1. `npx next build` + `tsc --noEmit` green (only the pre-existing `.test.ts` baseline errors).
2. Each of the 8 curated make pages (verify ≥3: `/aircraft/cessna`, `/aircraft/cirrus`,
   `/aircraft/piper`) renders a visible "{Make} — frequently asked questions" accordion
   with 3 Q&As, and the served HTML carries a `FAQPage` JSON-LD whose 3 answers appear
   verbatim in the visible DOM (Google parity).
3. A non-curated make with inventory (e.g. whatever `resolveMake` resolves that isn't in
   `SEO_MAKES`) renders NO FAQ and is otherwise unchanged (graceful — like the model pages).
4. The FAQ content is genuine + evergreen: no fabricated statistics, no live listing
   counts, make/brand-level (not a copy of the per-model FAQs).
5. QA smoke exit 0 at desktop 1280 + mobile 375 on the make pages (HTTP 200, zero
   app-origin console/hydration errors, zero horizontal overflow); screenshots look right.

## Out of scope
- Partnership make pages (`/partnerships/make/[make]`) and per-state FAQ variants — later slices.
- Any change to the existing per-model FAQs, listings, breakdown, JSON-LD, or layout.
