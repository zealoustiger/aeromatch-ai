indicator on partnership CARDS and the `/partnerships/[id]` DETAIL page, computed from

## Signals (verified against the live DB — all map to real `Partnership` columns)
1. **Real photo** — `images[0]` present AND `image_is_placeholder !== true` (not the make placeholder). 7/23 active.
2. **Complete specs** — `year` + `registration` + a price (`buy_in_price`|`monthly_fixed`|`hourly_wet`) + a real description (≥80 chars). 4/23.
3. **On-platform contact** — no `source_url` (an off-platform "View original post" redirect). 12/23.
4. **Member-posted** — `poster_id` is not null (a signed-up member, not a scrape). Field exists in schema; 0/23 today (all current listings are scraped). KEPT because the column is real, the signal is honest (it correctly reads 0 for scraped listings — the exact gap the human wants to close), and it is not fabricated. Max today is 2/4; full variation 0→2 across listings for QA.
