# Spec — Cost-explainer pillar guide

**Slug:** guide-cost · **Lane:** [goal] · **Branch:** night/guide-cost (off staging)

## Goal
Ship a SECOND, distinct content guide at `/guides/cost-of-aircraft-co-ownership`
targeting the high-intent query "How Much Does It Cost to Co-Own an Aircraft?" —
a genuine cost breakdown (NOT a near-duplicate of guide 1, which covers how it works).

## Scope (files expected to touch)
- NEW `src/app/guides/cost-of-aircraft-co-ownership/page.tsx` — mirrors guide 1's
  layout/metadata/breadcrumbs/FAQ-JSON-LD conventions; unique title/H1/meta/canonical/OG.
- EDIT `src/app/sitemap.ts` — add the new guide URL to the Guides static block.
- EDIT `src/components/Footer.tsx` — add the guide to the existing `guideLinks` list.
- EDIT `src/app/guides/aircraft-co-ownership/page.tsx` — add a real `<Link>` from
  guide 1 to guide 2 (wire the link graph both ways).

## Acceptance criteria
1. `/guides/cost-of-aircraft-co-ownership` renders real, readable, accurate cost
   content (not lorem/AI-slop): H1 + sectioned H2s covering buy-in & split,
   fixed/recurring vs variable/hourly costs, how shares split fixed vs hourly, and
   ≥1 worked EXAMPLE (Cessna 172, 3-way) with clearly-labeled ESTIMATE ranges
   ("typical ranges vary by region/aircraft"). Short FAQ. Honesty disclaimer; no
   legal/tax advice; no fabricated stats / fake precision.
2. Unique title + meta description + canonical + openGraph (mirroring the metadata
   shape of guide 1); title/H1 are DISTINCT from guide 1 (cost vs how-it-works).
3. Breadcrumbs (Home › Guides › {title}) via the existing Breadcrumbs component
   (emits BreadcrumbList JSON-LD).
4. Real internal `<Link>`s to: `/tools/cost-calculator`, `/tools/earnings-calculator`,
   guide 1 (`/guides/aircraft-co-ownership`), `/aircraft/cessna/172`, `/partnerships`.
   Guide 1 links back to guide 2. Added to footer Guides list. All resolve HTTP 200.
5. FAQPage JSON-LD that parses valid and matches the visible Q&As exactly (single
   source of truth); BreadcrumbList JSON-LD present. No fake review/rating markup.
6. In `/sitemap.xml` (resolves 200). `npx next build` + typecheck pass. QA against
   PRODUCTION build (`npm run start`): no console/hydration errors, no overflow at 375px.

## Out of scope
- NO schema / DB change. NO new /guides hub page (breadcrumb "Guides" stays plain
  text, matching guide 1). No new palette (sky-blue accent only). No changes to
  calculators or aircraft pages beyond the guide-1 backlink.
