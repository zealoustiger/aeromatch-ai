# Spec ‚Äî Listing comparison, SLICE 1 (partnerships)

**Lane:** `[want]` ¬∑ **Slug:** `compare` ¬∑ **Scoreboard at orient:** 54 (60 pageviews/7d)

## Goal
Let a user select 2‚Äì3 partnership listings on `/partnerships` and view them side-by-side
on a `/compare` page, using client-only selection state (no DB, no schema change).

## Scope (files expected to touch)
- NEW `src/components/CompareProvider.tsx` ‚Äî client context holding selected ids (React
  state synced to `sessionStorage`, max 3); exposes `toggle`, `remove`, `clear`, `isSelected`.
- NEW `src/components/CompareToggle.tsx` ‚Äî small "Compare" checkbox/toggle for each card.
- NEW `src/components/CompareTray.tsx` ‚Äî fixed bottom bar showing chosen listings +
  "Compare (N)" button + remove/clear.
- NEW `src/app/compare/page.tsx` ‚Äî server component, reads `?ids=a,b,c`, renders the
  side-by-side comparison table. `robots: { index: false }` (noindex). NOT in sitemap.
- NEW `src/components/CompareTable.tsx` (or inline) ‚Äî the responsive side-by-side table.
- EDIT `src/lib/partnerships.ts` ‚Äî add shared `getPartnershipsByIds(ids)` (reuse, no dup fetch).
- EDIT `src/app/partnerships/[id]/page.tsx` ‚Äî reuse the new `getPartnershipById` helper
  (de-dupe the private fetch) if clean to do so.
- EDIT `src/components/PartnershipCard.tsx` ‚Äî render `<CompareToggle>` (no-op outside provider).
- EDIT `src/app/partnerships/page.tsx` ‚Äî wrap listing area in `<CompareProvider>` + mount tray.

## Acceptance criteria
1. On `/partnerships`, each card shows a small "Compare" toggle; selecting it adds the
   listing to a fixed bottom **compare tray** that reflects the current selection (max 3;
   4th selection blocked with a hint). Tray has "Compare (N)", per-item remove, and clear.
2. Selection state is client-side only (React state + `sessionStorage`) ‚Äî NO DB / schema.
3. Clicking "Compare (N)" navigates to `/compare?ids=...`; the page renders a side-by-side
   table of the 2‚Äì3 partnerships with REAL fields only (aircraft, airport/location, share
   type, buy-in, monthly, wet rate, hours required, ratings, registration, status, link).
4. Each `/compare` column links to that listing's `/partnerships/[id]` (HTTP 200).
5. Graceful: 0/1 ids ‚Üí prompt to pick more; invalid/missing ids ‚Üí skipped; >3 ‚Üí capped at 3.
6. `/compare` has `robots: { index: false }` (noindex) and is NOT in the sitemap.
7. Mobile-first 375px: table usable (horizontal scroll or stacked), no overflow/clipping.
   Sky-blue accent only. No console/hydration errors. Build + typecheck green.
